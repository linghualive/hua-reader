# 沉浸式 RSS 阅读器 — 设计文档

## 项目概述

一款以沉浸式阅读为核心的 Android RSS 阅读器，帮助用户通过广泛阅读打破认知局限。

**核心理念**：用户至上，怎么看文章方便怎么来。

**技术栈**：React Native + React Native Paper (Material Design 3) + React Navigation + SQLite

**后端依赖**：自建 RSSHub 实例 (`http://linghua.icu:1200`)，同时支持任意标准 RSS/Atom 源

**目标平台**：Android

---

## 一、信息架构

### 底部导航（3 个 Tab）

| Tab | 名称 | 功能 |
|-----|------|------|
| 1 | 首页 | 刷文模式 / 列表模式切换浏览文章 |
| 2 | 发现 | 话题订阅管理、添加源、OPML 导入导出 |
| 3 | 我的 | 收藏文章、订阅管理、设置 |

### 页面流

```
首页（刷文/列表）
  ├── 刷文模式 ──上下滑动──→ 浏览文章预览
  │              ──点击/右滑──→ 全文阅读页 ──左滑──→ 返回
  │              ──双击──→ 收藏
  │
  └── 列表模式 ──话题Tab切换──→ 文章卡片列表
                ──点击卡片──→ 全文阅读页 ──左滑──→ 返回

发现
  ├── 话题订阅（内置推荐）
  ├── 添加 RSS 地址
  ├── 导入 OPML
  └── 导出 OPML

我的
  ├── 收藏文章列表
  ├── 我的订阅源管理（编辑/删除/分组）
  ├── 阅读统计（今日已读篇数等轻量数据）
  └── 设置
       ├── 主题色选择
       ├── 显示模式（亮色/暗色/纯黑AMOLED）
       ├── 阅读排版（字号/行距/阅读页背景色）
       ├── RSSHub 实例地址配置
       ├── 刷新频率
       └── 存储管理（缓存清理）
```

## 二、首页 — 双模式设计

### 2.1 刷文模式（默认）

全屏沉浸式，上下滑动切换文章，类似短视频的浏览体验。

**布局**：
```
┌─────────────────────────────────┐
│  ← 话题筛选 ▾        📋(列表)  │  ← 顶部半透明栏
│                                 │
│                                 │
│                                 │
│  文章标题（最多两行，大字醒目）   │
│                                 │
│  正文预览内容（3-5行关键段落）    │
│  提供足够信息让用户判断是否       │
│  值得深入阅读...                │
│                                 │
│                                 │
│  来源图标  36氪 · 2小时前       │
│                                 │
│  ────── 继续阅读全文 ──────     │  ← 明确的行动按钮
│                                 │
│            ⭐    ↗              │  ← 收藏 / 分享
└─────────────────────────────────┘
```

**交互**：
- 上滑：下一篇（ViewPager snap 动画）
- 下滑：上一篇
- 双击：收藏
- 点击"继续阅读全文"或右滑：进入全文阅读页
- 顶部话题筛选：下拉选择话题，过滤文章池
- 只展示未读文章，已读自动跳过

**智能排序**：
- 各话题文章交叉混排，避免连续出现同一来源
- 按时间倒序为基准

### 2.2 列表模式

传统信息流，适合有目的地浏览。

**布局**：
- 顶部：话题 Tab 栏（"全部"/"财经"/"科技"/"热榜"/...），横滑切换
- 内容：文章卡片列表，下拉刷新 + 上拉加载更多
- 右上角：切换按钮回到刷文模式

**文章卡片**：
```
┌─────────────────────────────────────┐
│ 来源图标  36氪 · 2小时前            │
│                                     │
│ 文章标题（最多两行）                  │
│ 摘要文字前两行内容预览...       [图] │
└─────────────────────────────────────┘
```

- 未读：标题加粗
- 已读：标题变灰
- 可切换"全部 / 仅未读"

## 三、全文阅读页（核心体验）

设计原则：**去除一切干扰，只留内容**。

### 3.1 内容获取策略

优先级：
1. RSS 全文（如果 RSS 源本身提供完整内容）
2. Readability 提取（通过 WebView 加载原文 → 提取正文 → 注入自定义 CSS）
3. 原始网页（兜底，提供"查看原文"按钮）

### 3.2 阅读页布局

```
┌─────────────────────────────────┐
│ ←               ⭐    ↗   ⋯   │  ← 顶部栏（滚动时自动隐藏）
│                                 │
│  文章标题                       │
│  来源 · 发布时间 · 预计阅读时间  │
│  ─────────────────────────      │
│                                 │
│  正文内容正文内容正文内容        │
│  正文内容正文内容正文内容        │
│  正文内容正文内容正文内容        │
│                                 │
│        [图片 / 自适应宽度]      │
│                                 │
│  正文内容正文内容正文内容        │
│  正文内容正文内容正文内容        │
│  ...                            │
│                                 │
│                                 │
├─────────────────────────────────┤
│  ■■■■■■■■■░░░  68%    Aa  🌙   │  ← 底部工具条（轻触唤出）
└─────────────────────────────────┘
```

### 3.3 排版参数（用户可调）

| 参数 | 范围 | 默认值 |
|------|------|--------|
| 字号 | 14-24sp | 17sp |
| 行距 | 1.4-2.0 倍 | 1.8 |
| 正文最大宽度 | — | 限制在合理阅读宽度内 |
| 阅读页背景色 | 白/米黄/浅灰/深色 | 跟随主题 |

### 3.4 阅读页交互

- 向下滚动：自动隐藏顶部栏和底部工具条，全屏沉浸
- 向上滚动 / 轻触屏幕中部：显示顶部栏和底部工具条
- 左滑：返回上一页
- 底部工具条：
  - 阅读进度条
  - `Aa` 按钮：弹出排版调节面板（字号/行距/背景色）
  - 夜间模式快捷切换
- 顶部 `⋯` 菜单：查看原文 / 复制链接 / 在浏览器打开

### 3.5 过渡动画

从刷文模式进入阅读页：Shared Element Transition，文章预览卡片"展开"为全文页面，用户感知是同一篇文章在展开而非跳转。

从列表模式进入：同样的展开动画。

返回：收缩回原位。

## 四、发现页 — 订阅管理

### 4.1 话题订阅（内置推荐）

话题卡片网格展示：

```
┌──────────┐ ┌──────────┐
│  💰 财经  │ │  📱 科技  │
│  8个源    │ │  7个源    │
│  [已订阅] │ │  [订阅]  │
└──────────┘ └──────────┘
┌──────────┐ ┌──────────┐
│  🔥 热榜  │ │  📰 新闻  │
│  3个源    │ │  5个源    │
│  [订阅]  │ │  [订阅]  │
└──────────┘ └──────────┘
```

**内置话题与订阅源映射表**（打包在 App 内，可远程更新）：

| 话题 | 包含的 RSSHub 路由 / RSS 源 |
|------|---------------------------|
| 财经 | /36kr/newsflashes, /wallstreetcn/news/global, /caixin/latest, /yicai/brief, /eastmoney/report/strategyreport, /gelonghui/live |
| 科技 | /sspai/index, /ithome/it, /solidot/www, /readhub/topic, /huxiu/article, /oschina/news |
| 热榜 | /zhihu/hot, /douban/movie/playing, /zhihu/daily |
| 新闻 | /thepaper/featured, /infzm/1, /36kr/information/web_news |
| 加密货币 | /jinse/lives |

点击话题卡片 → 展开显示具体源列表 → 可单独勾选/取消 → 一键全选订阅

### 4.2 手动添加

- 输入框粘贴 RSS/Atom URL
- App 自动探测并验证是否有效
- 验证成功后让用户选择归属话题（或创建新话题）

### 4.3 OPML 导入/导出

- 导入：从文件选择器选择 .opml 文件，解析后显示源列表，用户确认导入
- 导出：将当前全部订阅导出为 .opml 文件，保存到下载目录

## 五、我的 — 个人中心

### 5.1 收藏文章

- 独立列表展示所有收藏的文章
- 按收藏时间倒序
- 支持搜索收藏内容（标题 + 正文全文搜索）
- 左滑单条取消收藏
- 收藏文章永久保留，不受 7 天清理影响

### 5.2 我的订阅源

- 列表展示所有已订阅的源，按话题分组
- 每个源显示：名称、来源、最近更新时间、未读数
- 操作：
  - 长按拖拽调整排序
  - 左滑删除（二次确认）
  - 点击进入该源的专属文章列表
  - 编辑分组（移动到其他话题）
- 顶部"添加源"快捷入口

### 5.3 阅读统计（轻量）

- 今日已读 N 篇
- 本周已读 N 篇
- 连续阅读 N 天

### 5.4 设置

见下方主题系统和存储策略部分。

## 六、主题系统

### 6.1 全局主题色

基于 Material Design 3 Dynamic Color 系统：

- 预设 seed color：靛蓝、森绿、暖橙、玫红、紫罗兰、墨黑
- 选择一个 seed color 后自动生成完整配色方案（Primary / Secondary / Tertiary / Surface / OnSurface / Error 等）
- 所有组件统一使用 theme token，风格高度一致

### 6.2 显示模式

三种模式：
- 亮色模式
- 暗色模式
- 纯黑模式（AMOLED 省电）

### 6.3 阅读页独立配色

阅读页背景色独立于全局主题，可选：
- 纯白 (#FFFFFF)
- 米黄 (#FFF8E7) — 护眼暖色
- 浅灰 (#F5F5F5)
- 深色 — 跟随暗色/纯黑模式

## 七、数据架构

### 7.1 本地数据库（SQLite）

**表结构**：

```
feeds（订阅源）
├── id: 主键
├── title: 源名称
├── url: RSS/Atom 地址或 RSSHub 路由
├── source_type: 'rsshub' | 'native'
├── topic_id: 所属话题
├── icon_url: 来源图标
├── last_fetched: 上次拉取时间
└── created_at

topics（话题）
├── id: 主键
├── name: 话题名称
├── icon: 图标标识
├── sort_order: 排序
└── is_builtin: 是否内置话题

articles（文章）
├── id: 主键
├── feed_id: 所属订阅源
├── guid: RSS guid（去重用）
├── title: 标题
├── summary: 摘要
├── content: 全文内容（Readability 提取后缓存）
├── url: 原文链接
├── image_url: 缩略图
├── published_at: 发布时间
├── fetched_at: 拉取时间
├── is_read: 是否已读
├── is_bookmarked: 是否收藏
└── bookmarked_at: 收藏时间

settings（设置）
├── key
└── value
```

### 7.2 文章生命周期

```
拉取新文章 → 存入 articles 表（is_read=false, is_bookmarked=false）
  → 用户阅读 → is_read=true
  → 用户收藏 → is_bookmarked=true, bookmarked_at=当前时间
  → 7 天后自动清理（条件：is_bookmarked=false）
  → 收藏文章永久保留，除非用户手动删除
```

### 7.3 订阅源数据流

```
App 启动 / 手动下拉刷新 / 定时后台刷新
  → 遍历所有订阅源
  → RSSHub 源：请求 http://linghua.icu:1200/{route}
  → 原生 RSS 源：直接请求 RSS URL
  → 解析 XML（RSS 2.0 / Atom）
  → 按 guid 去重，新文章插入 articles 表
  → 通知 UI 更新
```

### 7.4 全文获取流程

```
用户点击"继续阅读全文"
  → 检查 articles.content 是否已缓存
  → 有缓存 → 直接渲染
  → 无缓存 → 加载原文 URL
     → 使用 Readability 算法提取正文
     → 缓存到 articles.content
     → 注入自定义 CSS 渲染
  → 提取失败 → 降级到 WebView 加载原始网页
```

## 八、交互设计规范

### 8.1 手势一览

| 场景 | 手势 | 动作 |
|------|------|------|
| 刷文模式 | 上滑 | 下一篇文章 |
| 刷文模式 | 下滑 | 上一篇文章 |
| 刷文模式 | 双击 | 收藏文章 |
| 刷文模式 | 点击"继续阅读" / 右滑 | 进入全文阅读 |
| 阅读页 | 向下滚动 | 隐藏顶栏底栏，沉浸阅读 |
| 阅读页 | 向上滚动 / 点击中部 | 显示顶栏底栏 |
| 阅读页 | 左滑 | 返回上一页 |
| 列表模式 | 下拉 | 刷新 |
| 列表模式 | 点击卡片 | 进入全文阅读 |
| 收藏列表 | 左滑单条 | 取消收藏 |
| 订阅管理 | 长按拖拽 | 调整排序 |
| 订阅管理 | 左滑 | 删除订阅源 |

### 8.2 动画规范

- 页面转场：300ms，ease-in-out
- 刷文翻页：ViewPager snap 弹性动画
- 阅读页进入：Shared Element Transition（卡片展开为全文）
- 收藏反馈：⭐ 图标缩放弹跳 + 短震动
- 下拉刷新：Material Design 3 标准 pull-to-refresh
- 所有动画遵循 Material Motion 规范，保持风格统一

### 8.3 空状态设计

| 场景 | 展示 |
|------|------|
| 首次使用，无订阅 | 引导去"发现"页订阅话题 |
| 话题下无文章 | "暂无内容，下拉刷新试试" |
| 所有文章已读（刷文模式） | "已经刷完了，去发现更多话题？" |
| 收藏为空 | "还没有收藏，阅读时双击可收藏" |
| 网络异常 | "网络不给力，点击重试" + 显示本地缓存内容 |

## 九、性能要求

- 文章列表使用 FlatList / FlashList 虚拟化渲染
- 图片使用 react-native-fast-image 懒加载 + 缓存
- 数据库操作异步执行，不阻塞 UI
- 刷文模式预加载前后各 2 篇文章内容
- 首屏加载时间 < 1 秒
- 后台定时刷新使用 React Native Background Fetch
